# Hot Monitor - 需求文档

## 1. 项目概述

Hot Monitor 是一个轻量级的实时热点监控工具，帮助用户第一时间获取指定领域的热点信息（如 AI 大模型更新），避免人工手动搜索，通过自动化手段发现热点变化并及时通知用户。

## 2. 核心功能

### 2.1 关键词监控（Alert 模式）

- 用户手动输入要监控的关键词（如 "GPT-5"、"Claude 4"）
- 系统定时从多个信息源搜索该关键词相关内容
- 利用 AI 识别内容真伪，过滤标题党、假冒信息
- 当发现真实有效的相关内容时，**立即发送通知**给用户

### 2.2 热点发现（Discovery 模式）

- 用户输入指定范围（如 "AI编程"、"大模型"）
- 系统每隔一段时间自动搜集该范围内的热点
- AI 对热点进行提取、排序、摘要
- 用户可在页面上查看热点列表

### 2.3 通知系统

- 浏览器原生通知（Browser Notification API）
- 页内通知面板（未读消息徽章 + 通知列表）
- 通知声音提示

## 3. 产品形态

### 3.1 Web 页面（优先开发）

- 响应式设计，兼容桌面端和移动端
- **赛博朋克监控站**视觉风格：深色基底 + 霓虹渐变 + 终端/雷达动效
- Bento Grid 布局，信息密度高

### 3.2 Agent Skills 技能包（后续开发）

- 封装为标准 Agent Skills，可被其他 AI Agent 调用
- 提供热点查询、关键词设置、监控启停等能力

## 4. 信息源

| 序号 | 信息源 | 获取方式 | 说明 |
|------|--------|---------|------|
| 1 | 网页搜索 | 后端爬虫抓取搜索引擎结果页 | 无需 API，注意频率控制 |
| 2 | Twitter/X | TwitterAPI.io REST API | 第三方 Twitter API，按量付费 |
| 3 | RSS 订阅 | 解析 RSS/Atom Feed | 订阅科技媒体 RSS 源 |

## 5. 非功能需求

- 轻量敏捷，不过度工程化
- 本地 JSON 文件存储，无需数据库
- 前端页面独特、有辨识度
- 搜索爬虫需控制频率，避免被封禁
